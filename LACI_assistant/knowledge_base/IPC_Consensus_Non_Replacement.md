# IPC Consensus Non-Replacement Axiom for LACI

**Status:** user-confirmed LACI governance anchor.

## Rule

No model-generated output, including IPC Outcome Modeling, IPC-Indicative outputs, proxy-trained food security outputs, AMN models, alerts, thresholds, scores, phase-like labels, or affected-population estimates, can ever replace the consensus-based IPC classification process or become the actual IPC outcome by itself.

This is not a judgment to be made during Learn, Assess, Calibrate, Integrate, or the IPC Analysis Use Checklist. It is a standing premise of LACI.

## What LACI Should Assess Instead

LACI should assess only whether a model-generated output can support a bounded role, such as:

- documenting model metadata and target provenance;
- informing technical review;
- prompting further investigation;
- supporting projection-assumption review;
- supporting contextual risk monitoring;
- informing triangulation planning;
- documenting why an output should not be considered;
- defining non-use, labeling, and communication guardrails.

## Flow Implication

Do not ask: "Can this model replace IPC classification?"

Ask instead:

- What bounded IPC workflow role, if any, is this output eligible to support?
- What evidence, geography, time-window, performance, and context checks are required before analysts may consider it?
- What must analysts not do with the output?
- What labeling and access controls prevent the output from being mistaken for IPC consensus classification, AMN classification, famine classification, or official population/caseload estimates?
- What reassessment triggers would downgrade or block consideration?

## Template Implication

Assess Card Part 3 should focus on protocol fit, target provenance, convergence risk, threshold misuse risk, and categorical non-use rules. It should not ask reviewers to decide whether a model can replace analyst-led IPC consensus, because the answer is always no.
