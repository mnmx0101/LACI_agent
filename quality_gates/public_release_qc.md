# Public Release Quality Gate

Use together with `_system/PUBLIC-SAFETY-CHECKLIST.md`.

- [ ] **Repo-relative links:** Links do not point to local machine paths.
- [ ] **No private files:** Raw notes, operational outputs, and unpublished materials are excluded.
- [ ] **No stale names:** Public files use `LACI_agent` consistently.
- [ ] **No unsupported role language:** Ownership language is clear, diplomatic, and consistent.
- [ ] **Examples are public-safe:** Any example committed to the repo is safe to share publicly.
- [ ] **Markdown links pass:** Relative markdown links resolve inside the repo.
- [ ] **README path is clear:** New users can find entrypoints, prompts, intake forms, runbooks, and quality gates.
