# LACI_agent Navigation

Use this file when you want to explore the repo before starting a workflow.

## Start Here

- **Understand LACI:** Read [README.md](README.md), [CORTEX.md](CORTEX.md), and [core_documents/LACI_Overview.md](core_documents/LACI_Overview.md).
- **Understand user roles:** Read [ROLE_ROUTER.md](ROLE_ROUTER.md) and [ENTRY.md](ENTRY.md).
- **Browse templates:** Open [templates/](templates/).
- **Browse knowledge base / wiki:** Open [knowledge_base/](knowledge_base/).
- **Review workflow guidance:** Open [workflows/](workflows/) and [runbooks/](runbooks/).
- **Start from a model paper:** Use [prompts/learn_card_from_model_paper.md](prompts/learn_card_from_model_paper.md).
- **Verify or update a card:** Use [skills/LACI_Model_Builder_Reviewer.md](skills/LACI_Model_Builder_Reviewer.md), [prompts/assess_card_from_learn_card.md](prompts/assess_card_from_learn_card.md), and the relevant quality gate.
- **Prepare checklist inputs:** Use [prompts/use_checklist_from_operational_note.md](prompts/use_checklist_from_operational_note.md) only when the use case definition, latest operational note, and current context are available.

## Current Scope Note

`LACI_agent` is currently strongest for **Learn** and **Assess** work. It also includes early support for **Calibrate** and **Integrate**, but final checklist use and final verdicts require operational notes, use case definitions, current context, and human ownership.
